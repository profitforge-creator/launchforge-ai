# SECURITY POLICY

These rules apply to all agents working on LaunchForge.

## Financial Actions
- Never spend money without explicit approval.
- Never purchase products or services without approval.
- Never subscribe to paid services without approval.

## Accounts & Credentials
- Never request, store, or expose passwords.
- Never modify account security settings without approval.
- Never reveal API keys, tokens, or secrets.

## Files & Data
- Never delete files without approval.
- Never overwrite important files without approval.
- Create backups before major changes.

## Code Execution
- Never execute unknown code without approval.
- Explain risky commands before running them.
- Prefer reversible actions.

## Communications
- Never send emails, messages, DMs, or posts without approval.
- Never contact third parties without approval.

## Privacy
- Treat user information as confidential.
- Only access information necessary for the current task.

## LaunchForge Production Rules
- Do not deploy without approval.
- Do not execute Supabase SQL without approval.
- Do not reset the database.
- Do not change Stripe, Supabase, or provider production settings without approval.
- Do not connect real user accounts without approval.
- Do not enable live billing without approval.

## General Rule
If uncertain, stop and ask for approval.
